Executive summary

Archive Detective turns Library of Congress Chronicling America newspaper fragments into playable micro-mysteries. Players read real OCR text and clippings, follow branching leads, then reveal what is archive fact vs editorial glue.

5Playable cases
12LOC snippets ingested
~5.4kLines (src + scripts + cases)

The shipping UI is a custom gradio.Server board (HTML/CSS/JS) — not default Gradio Blocks. Prebuilt case JSON means the demo runs fully offline.

LOC ingest pipeline

  1. Fetch — scripts/fetch_chronicling_america.py pulls snippets + IIIF images into data/raw/snippets/
  2. Rank — ingest/ranking.py scores OCR for mystery signals (questions, secrets, proper nouns, noise)
  3. Build packs — clue_builder.py heuristics or optional MiniCPM-V / llama.cpp JSON extraction
  4. Case graph — build_cases_from_ingest.py emits branching JSON in data/cases/

Hero demos

  • The Hart Notice (georgetown_notice) — synthetic passphrase notice; teaches coded-message mechanics
  • The Midnight Brief (1937 Aimee McPherson organ signal) — real LOC OCR + scan; highest mystery score

gr.Server custom board

We replaced Gradio Blocks with gradio.Server + static board. The frontend calls /gradio_api/call/{api} and parses SSE data: lines — same contract Gradio uses internally.

main.py → server_app.build_server()
  GET  /              index.html
  GET  /board.css     /board.js
  GET  /assets/*      clipping scans
  API  catalog | start_case | choose_lead | reveal

Design direction

Warm monochrome detective desk: cork panels, newsprint cards, brass labels, wax-seal primary actions. Case picker shows titles (★ for hero cases), inline loading on lead follow, scroll-to-reveal animation.

Mistakes we hit

  • Deployed to HF Space before local browser QA — user blocked further deploy until --confirm-local
  • Static files lived under src/archive_detective/static/board/ but early docs referenced static/board/ at repo root
  • Gradio SSE poll format is easy to get wrong; must parse data: [...] not raw JSON
  • Premature Modal/MiniCPM path added complexity; prebuilt cases were the right MVP

Local llama.cpp path

./scripts/run_llama.sh          # llama-server on :8080

Recommended GGUF: Qwen2.5-3B-Instruct-Q4_K_M (~2GB). Text-from-OCR extraction only — images stay as LOC scans; no cloud vision API required.

LocateAnything-3B (future fit)

LocateAnything-3B grounds text queries on document images — a natural next step for pointing at which line of a noisy clipping supports each evidence card. Not integrated for hackathon deadline; would replace heuristic entity pinning with bbox highlights on the cork board.

Lessons

  • Ship prebuilt content first; wire live models second
  • Custom UI on gr.Server is viable for Off-Brand without ejecting from HF Spaces
  • LOC OCR is already a game mechanic — lean into noise, don't over-clean
  • Always run verify_local.py + browser click-through before deploy_space.py
